# HanGarden PoC 과제 요약

## 개요

HanGarden은 한국수목원정원관리원의 식물 데이터와 생성형 AI를 활용하여, 관람객 맞춤형 식물해설 콘텐츠를 생성하고 관리하는 콘텐츠 플랫폼 PoC이다. 내부 프로젝트명은 PlantPlanet이며, 수목원 현장에서 식물 표찰 QR, 모바일 웹, 키오스크, 정적 포스터 등 다양한 단말 환경에 맞는 해설 콘텐츠를 빠르게 제작하고 실증하는 것을 목표로 한다.

## 프로젝트 목표

본 프로젝트의 핵심 목표는 수목원이 보유한 식물 정보와 해설 자산을 기반으로, 관람객의 연령, 언어, 관심사, 방문 시점, 전시 공간, 콘텐츠 목적에 맞는 맞춤형 디지털 해설 콘텐츠를 생성하는 것이다.

기존 수목원 콘텐츠는 식물 정보와 해설 자료는 풍부하지만, 관람객별 맥락에 맞춰 실시간으로 변형하거나 다양한 전시 단말에 맞게 재구성하는 체계가 부족하다. HanGarden PoC는 생성형 AI를 통해 이 문제를 해결하고, 수목원 운영자가 직접 콘텐츠를 생성, 검수, 관리할 수 있는 구조를 검증한다.

## 과제 목적

이 PoC는 다음 목적을 가진다.

- 생성형 AI 기반 식물해설 콘텐츠 생성 가능성 검증
- 식물 DB와 생성 설정 JSON을 결합한 콘텐츠 제작 흐름 검증
- 소개형, 스토리텔링형, 퀴즈형, 미션형 등 다양한 콘텐츠 유형 실험
- 모바일, 키오스크, 정적 포스터 등 배포 단말별 레이아웃 생성 검증
- 로컬 서버와 Cloudflare Tunnel을 활용한 외부 접속형 시연 환경 구성
- Codex CLI를 백엔드 생성 엔진으로 연동하여 React 콘텐츠 페이지 자동 생성 검증

## 현재 PoC 동작 방식

관리자는 React 기반 프론트엔드에서 콘텐츠 생성 마법사를 사용한다. 생성 절차에서는 대상 식물, 콘텐츠 목적, 콘텐츠 유형, 배포 단말, 관람 대상, 언어, 계절, 현장 위치, 강조 주제, 문장 톤, 추가 요청사항 등을 선택한다.

현재 실제 한수정 DB는 연결되어 있지 않으며, `frontend/public/mock_db.json`에 있는 목업 식물 데이터를 사용한다. 사용자가 생성 요청을 완료하면 프론트엔드는 설정값과 콘텐츠 메타데이터를 백엔드 WebSocket `/ws/generate`로 전송한다.

백엔드는 다음 순서로 동작한다.

1. 생성 요청 JSON과 콘텐츠 메타데이터를 수신한다.
2. 콘텐츠 정보를 `generated/contents.json`에 저장한다.
3. 생성 작업 상태를 `generated/generation-jobs.json`에 기록한다.
4. 요청 원본을 `generated/layout-jobs/`에 저장한다.
5. `scripts/generate-layout-page.ts`를 실행한다.
6. 해당 스크립트가 Codex CLI를 호출하여 React TSX 콘텐츠 페이지를 생성한다.
7. 생성된 파일은 `frontend/src/generated/layouts/`에 저장된다.
8. `layout-manifest.json`과 `generatedLayoutRegistry.tsx`를 갱신한다.
9. 프론트엔드 빌드를 실행하여 생성된 페이지가 타입 및 번들 검증을 통과하는지 확인한다.
10. 성공 시 클라이언트에 완료 응답을 보내고, 콘텐츠는 `#/content/{id}` 경로로 접근 가능해진다.

## 시연 및 접속 구조

로컬 개발 시에는 프론트엔드와 백엔드를 각각 실행할 수 있다.

- Frontend: `http://localhost:5173`
- Backend: `http://localhost:4000`

외부 시연을 위해서는 다음 명령을 사용한다.

```bash
npm run start:public
```

이 명령은 프론트엔드와 백엔드를 빌드하고, 백엔드 서버를 실행한 뒤 Cloudflare Quick Tunnel을 열어 외부 접속 URL을 생성한다. 이후 별도 `hangarden` 리디렉트 저장소의 `redirect-target.json`을 갱신하여 고정 진입 주소에서 최신 Cloudflare Tunnel 주소로 연결되도록 한다.

## 현재 PoC 범위

현재 구현된 범위는 다음과 같다.

- HanGarden 관리자 대시보드
- 콘텐츠 생성 마법사
- 목업 식물 DB 기반 대상 식물 선택
- 생성 설정 JSON 구성
- 기본 레이아웃 저장 방식
- Codex CLI 기반 생성형 React 레이아웃 생성 방식
- 생성 콘텐츠 목록 및 상세 조회
- 생성 작업 상태 관리
- 생성된 콘텐츠의 개별 라우팅
- AI 생성 콘텐츠 수정 요청 흐름
- 전시 관리 화면의 기초 구조
- Cloudflare Tunnel 기반 외부 시연 환경

## 향후 확장 방향

향후에는 목업 DB를 실제 한수정 식물 DB와 연동하고, 콘텐츠 검수 및 승인 워크플로우, 전시 단말 배포 관리, 사용자 반응 데이터 수집, 음성 해설, 다국어 콘텐츠 생성, 유사 식물 추천, 현장 QR 기반 관람 동선 연결 등으로 확장할 수 있다.

이 PoC는 단순한 정적 페이지 생성 도구가 아니라, 수목원 운영자가 현장 맥락에 맞는 디지털 해설 콘텐츠를 직접 생성하고 관리할 수 있는 생성형 AI 콘텐츠 운영 플랫폼의 초기 검증 모델이다.
