# kiosk 레이아웃 규칙

`deploymentUse: "kiosk"`는 1920x1080 FHD 가로형 전시 디스플레이입니다. 방문자는 마우스/터치로 짧게 상호작용할 수 있지만, 콘텐츠 핵심 경험은 첫 화면 안에서 완결되어야 합니다.

## 목표

- 16:9 FHD 화면에서 제목, 핵심 이미지, 핵심 설명, 주요 상호작용이 한 화면에 보입니다.
- 정보량을 줄여서 맞추며, CSS로 잘라서 맞추지 않습니다.
- 페이지 목적은 멀리서도 3초 안에 이해되어야 합니다.

## 필수 레이아웃 규칙

- 루트 `article`, `main`, 최상위 `div`에 `height: "100vh"`, `height: "100dvh"`, `height: "100%"`를 쓰지 않습니다.
- 루트 `article`, `main`, 최상위 `div`에 `overflow: "hidden"`을 쓰지 않습니다.
- 텍스트를 담는 카드, 섹션, 리스트, 설명 영역에 `overflow: "hidden"` 또는 `maxHeight`로 내용을 자르는 구조를 만들지 않습니다.
- `overflow: "hidden"`은 이미지 크롭용 media wrapper처럼 잘려도 의미 손상이 없는 시각 영역에만 제한적으로 사용합니다.
- FHD 안에 들어가지 않을 정도로 길어지면 섹션 수와 문장 수를 줄입니다.
- 주요 그리드는 2~3열 이내로 구성하고, 카드 수는 한 화면에 읽을 수 있는 수준으로 제한합니다.
- 문장은 짧게 압축하고, 긴 문단 대신 핵심 문장과 짧은 보조 설명을 사용합니다.
- `position: absolute`는 이미지 캡션, 장식, 배지에만 사용하고, 본문 텍스트 배치에는 사용하지 않습니다.
- 텍스트가 이미지나 카드 경계를 벗어나지 않도록 `boxSizing: "border-box"`, `minWidth: 0`, `wordBreak: "keep-all"` 또는 적절한 줄바꿈 처리를 사용합니다.

## 금지 패턴

- `height: "100dvh"`와 `overflow: "hidden"`을 함께 사용해 화면을 강제로 자르는 구조
- `height: "57dvh"`처럼 상단/하단을 vh 합산으로 딱 맞춘 뒤 내부 텍스트가 넘치는 구조
- 긴 설명을 만든 뒤 `maxHeight`나 `overflow: hidden`으로 숨기는 구조
- 모바일처럼 세로로 긴 스크롤 페이지를 만든 뒤 키오스크라고 주장하는 구조

## 자체 점검

- 작성 후 최상위 레이아웃에 `height: "100vh"`, `height: "100dvh"`, 루트 `overflow: "hidden"`이 남아 있으면 수정합니다.
- 텍스트/카드 하단이 잘릴 가능성이 있으면 CSS로 숨기지 말고 콘텐츠 양을 줄입니다.
- 1920x1080 한 화면에서 실제 방문자가 읽어야 하는 내용이 모두 보이는지 점검합니다.
