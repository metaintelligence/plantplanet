# staticPoster 레이아웃 규칙

`deploymentUse: "staticPoster"`는 상호작용 없는 한 페이지 포스터형 React 렌더링입니다. 방문자는 버튼을 누르지 않고 한 장의 안내물처럼 읽습니다.

## 목표

- 한 장의 포스터처럼 제목, 대표 이미지, 핵심 메시지, 보조 정보가 한 화면에 정리됩니다.
- 퀴즈 정답 공개, 단계 진행, 클릭 인터랙션처럼 조작이 필요한 요소는 만들지 않습니다.
- 정보 밀도를 높이되 텍스트가 빽빽하거나 잘리지 않게 합니다.

## 필수 레이아웃 규칙

- 루트 요소에 `overflow: "hidden"`을 사용해 콘텐츠를 자르지 않습니다.
- 포스터 비율 안에 들어가지 않으면 문장 수와 카드 수를 줄입니다.
- 버튼, 탭, 퀴즈 선택지, 토글처럼 상호작용을 전제로 하는 UI를 만들지 않습니다.
- CTA가 필요하다면 버튼이 아니라 짧은 안내 문구나 QR 배치 영역처럼 정적으로 표현합니다.
- 이미지는 대표성 있게 크게 쓰되, 본문을 밀어내지 않도록 안정적인 비율을 사용합니다.
- 긴 문단보다 제목, 짧은 설명, 2~4개의 핵심 포인트를 우선합니다.

## 금지 패턴

- `useState` 기반 인터랙션
- 클릭해야만 내용이 보이는 구조
- 페이지 하단 정보를 `overflow: hidden`으로 숨기는 구조
- 모바일 스크롤 페이지처럼 길게 이어지는 구조

## 자체 점검

- 모든 텍스트가 포스터 안에서 한 번에 읽히는지 확인합니다.
- 상호작용 없이도 내용이 완결되는지 확인합니다.
- 정보가 많으면 숨기지 말고 줄입니다.
