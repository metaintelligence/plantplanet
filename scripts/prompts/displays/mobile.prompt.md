# mobile 레이아웃 규칙

`deploymentUse: "mobile"`은 세로형 모바일 브라우저입니다. 방문자는 터치로 조작하며, 세로 스크롤은 허용됩니다.

## 목표

- 좁은 화면에서 읽기 쉽고, 손가락으로 누르기 편한 콘텐츠를 만듭니다.
- 한 번에 모든 정보를 보여주려 하지 않고, 자연스러운 세로 흐름으로 구성합니다.
- 가로 스크롤이 생기지 않아야 합니다.

## 필수 레이아웃 규칙

- 루트는 `width: "100%"`, `minHeight` 중심으로 구성하고 고정 폭을 쓰지 않습니다.
- `minWidth: 0`, `boxSizing: "border-box"`, `maxWidth: "100%"`를 적극 사용해 가로 넘침을 방지합니다.
- `height: "100vh"` 또는 `height: "100dvh"`로 전체 페이지 높이를 고정하지 않습니다.
- 모바일에서는 세로 스크롤을 허용하므로 루트나 주요 섹션에 `overflow: "hidden"`으로 내용을 자르지 않습니다.
- 버튼과 선택지는 터치하기 쉬운 크기와 충분한 간격을 둡니다.
- 그리드는 모바일 폭에서 1열로 전환되어야 합니다.
- 이미지와 미디어는 `width: "100%"`, `height: "auto"` 또는 안전한 aspect ratio 기반으로 렌더링합니다.

## 금지 패턴

- 1920px 같은 고정 폭 레이아웃
- 여러 열이 모바일에서도 유지되어 텍스트가 찌그러지는 구조
- 텍스트가 긴데 카드 높이를 고정하고 숨기는 구조
- 버튼이 너무 작거나 서로 붙어 있는 구조

## 자체 점검

- 360px 전후의 모바일 폭에서도 가로 스크롤 없이 읽히는지 점검합니다.
- 긴 단어와 학명 때문에 레이아웃이 깨지지 않는지 점검합니다.
- 상호작용 요소가 터치 가능한 크기인지 점검합니다.
